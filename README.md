# カメラ補正比較アプリケーション 📷

補正ありとなしの写真を比較するPWAアプリケーション

## 機能 ✨

- **補正設定制御**: Camera APIを使用して以下の補正をカントロール
  - whiteBalanceMode（手動・色温度6500K）
  - exposureMode（手動）
  - focusMode（自動）
  - zoom制御

- **並列撮影**: 補正なしとデフォルト補正の2枚を同時に撮影

- **カラー分析**: RGB値と明るさを自動比較

- **ダウンロード機能**: 撮影画像をJPEG形式でダウンロード

- **複数カメラ対応**: デバイス選択機能搭載

- **解像度選択**: 640x480, 1280x720, 1920x1080対応

- **PWA対応**: オフライン動作、ホーム画面追加可能

## 使用方法 🚀

### ブラウザでの起動

1. HTTPSサーバーで提供します（Camera APIはHTTPSすまたはLocalhostが必須）
2. アプリを開く
3. カメラアクセスを許可
4. 撮影ボタンで写真をキャプチャ

### PWAとしてインストール

1. ブラウザの「アプリをインストール」をクリック
2. ホーム画面にアプリアイコンが追加されます
3. オフライン時も起動可能

## 補正の詳細

### 補正なし（デフォルト設定）

```javascript
advanced: [{
    whiteBalanceMode: 'manual',
    colorTemperature: 6500,      // 昼光の色温度（ケルビン）
    exposureMode: 'manual',
    exposureCompensation: 0,
    exposureTime: 33333,
    focusMode: 'continuous',
    zoom: 1.0
}]
```

### デフォルト補正

ブラウザのデフォルト設定のCamera APIを使用

## 色分析について 📊

比較ビューでは以下の項目を分析：

- **赤値 (R)**: 0-255の赤成分の平均値
- **緑値 (G)**: 0-255の緑成分の平均値
- **青値 (B)**: 0-255の青成分の平均値
- **明るさ**: (R+G+B)/3 の平均値

差分はパーセンテージで表示

## トラブルシューティング 🔧

### カメラへのアクセスが拒否される
1. ブラウザの権限設定を確認
2. HTTPS または localhost で実行
3. HTTPSの場合は自己署名証明書も可

### Camera API補正設定が反映されない
- デバイスが補正設定をサポートしていない可能性
- フォールバックでデフォルト設定を使用
- ブラウザコンソールで警告メッセージを確認

### 撮影画像が真っ暗
1. カメラのレンズをクリア
2. 照明条件を改善（室内の場合）
3. 別のアプリでカメラテストを実施

## サポート 💬

問題が発生した場合：
1. ブラウザのコンソール（F12）でエラーを確認
2. `service-worker.js` が正しくロードされているか確認
3. デバイスキャッシュをクリアして再度試行
